Trabajo: Introducción a Git y GitHub

    Conceptos Fundamentales

  1. Control de versiones: Antes de conocer Git, ¿cómo gestionabas las diferentes versiones de tus proyectos o trabajos? ¿Qué problemas enfrentabas con ese método? ¿Cómo crees que Git puede resolver esos problemas?

    Antes de conocer git, no tenia una manera clara de administrar las versiones de mi proyecto, lo que hacia era tener un archivo y clonarlo para asi poder experimentar sin temor. El problema que generaba esto, es que era muy tardado, perdiendo mucho tiempo. Creo que GitHub puede ayudarme a controlar las versiones de mi proyecto de una manera mucho mas eficiente y sin perder tiempo.

  2. Commits significativos:El curso enfatiza la importancia de hacer commits con mensajes descriptivos. ¿Por qué crees que es importante documentar claramente cada cambio que realizas en un proyecto? Piensa en un ejemplo de tu vida académica donde una mejor documentación te habría ahorrado tiempo o confusión.

    Esto es importante ya que una buena documentacion puede ayudar a que tus compañeros de trabajo puedan comprender rapidamente lo que desarrollaste, y asi ahorrar tiempo y tener una idea precisa de los cambios. Un ejemplo de esto en mi vida academica puede ser en la funcionalidad de un selector de archivos/imagenes, esto al no estar bien documentado, se generaron confusiones en que tipo de archivo podria seleccionar, o que deberia hacer al seleccionarse, etc.

  3. Diferencia Git vs GitHub:Explica con tus propias palabras la diferencia entre Git y GitHub. ¿Por qué necesitamos ambas herramientas? ¿Podrías usar Git sin GitHub?

    Por lo que entendi en el curso, Git es un controlador de versiones de manera local, y GitHub que tambien es un controlador de versiones, pero con la diferencia que este esta alojado en una nube, en otras palabras, Git es utilizado en tu computadora, sin necesidad de internet ni nada externo, pero perdiendo la facilidad de compartir las versiones con compañeros de Trabajo, en cambio GitHub al estar en una nube, obligatoriamente necestia internet, pero este si tiene la facilidad de compartir las versiones con tus compañeros de trabajo.
    Es necesario contar con ambas para que si ocurre una dificultad con la conexion de internet, puedas seguir trabajando sin ningun tipo de problema, y GitHub para tener una copia del trabajo, por si se da el caso de un fallo del dispositivo utilizado, y asi no perder tu progreso.
    Si, se puede usar Git sin GitHub, pero como lo ya mencionado antes, al hacerlo, perderemos la facilidad de compartir las versiones del proyecto a tus compañeros de trabajo, y no dispondremos de una copia en la nube, por si falla nuestro dispositivo de trabajo.

    4. Aplicación Práctica

  4. Branches (ramas):El concepto de trabajar en ramas paralelas puede parecer confuso al principio. ¿En qué situaciones de tu vida estudiantil o personal te gustaría poder "crear una rama" para experimentar sin afectar tu "rama principal"? Describe un ejemplo concreto.

    Me gustaria crear una rama secundaria para poder experimentar y/o optimizar partes de un proyecto, por ejemplo: El año pasado cree un sistema para una biblioteca utilizando C#, en este proyecto tuve que probar muchas cosas nuevas, e ir probando y cambiando el codigo repetidamente. Con Git/GitHub podria haber guardado un progreso funcional y de esa base crear una rama e ir experimentando, y asi, aunque genere cambios, siempre tendria el codigo funcional a disposicion.

  5. Colaboración remota: GitHub permite que personas de diferentes partes del mundo colaboren en el mismo proyecto. ¿Qué ventajas y desafíos identificas en este tipo de colaboración? ¿Cómo crees que esto cambia la forma de trabajar en equipo comparado con trabajar todos en el mismo lugar físico?

    La ventaja principal que identifico es que no deben estar todos en el mismo sitio para trabajar en conjunto, y que no estan obligados a trabajar todos al mismo tiempo, uno puede trabajar de 8 AM a 2 PM y los demas a las 2 PM a 7 PM sin tener ningun tipo de problemas
    Esto aumenta la comodidad de los trabajadores al no tener que estar en el mismo sitio, ni trabjar en el mismo horario, cada uno podria decidir cuando quiere trabajar.

  6. Resolución de conflictos: Cuando dos personas modifican el mismo archivo, Git detecta un "conflicto" que debe resolverse manualmente. ¿Qué habilidades personales y técnicas crees que necesitas desarrollar para resolver conflictos de manera efectiva en un equipo de desarrollo?

    Yo creo que para solucionar este conflicto deberian tener una buena comunicacion para poder distinguir cual rama trabaja cierto desarrollador y cual el otro desarrollador, y asi no modificar una rama erronea.

    7. Reflexión Profesional

  7. Portafolio digital: Tu perfil de GitHub funciona como un portafolio público de tu trabajo. ¿Qué tipo de proyectos te gustaría tener en tu perfil para mostrar tus habilidades? ¿Cómo crees que esto puede influir en tus oportunidades laborales futuras?

    Por el momento me gustaria tener un perfil orientado en el desarrollo de paginas/aplicaciones WEB.A mi parecer, esto puede beneficiar bastante a los que estamos recien entrando al mundo de la programacion, al poder mostrar con facilidad a los demas nuestros proyectos creados, y asi demostrar nuestra forma y habilidades de trabajo.
